# NexVote - Decentralized Election Platform 🗳️

NexVote is a fully functional, highly advanced decentralized election application built with Solidity, Foundry, and Vanilla JavaScript. It features a stunning "Cyberpunk/Terminal" themed UI and powerful automation scripts to simplify the deployment pipeline. 

This project empowers administrators (the contract owners) to instantiate multiple elections with unique candidates while allowing users to securely and immutably cast their votes on the EVM blockchain.

## ✨ Core Features

- **Smart Contracts (Solidity)**:
  - `ElectionFactory` contract manages multiple concurrent elections.
  - Prevents double voting ensuring a strict "One person, one vote" mechanism.
  - Enterprise-grade **NatSpec documentation** comments applied throughout.
  - Highly optimized bytecode via internal function extraction for modifiers.
- **Cyberpunk Frontend App (Vanilla JS & Ethers.js v6)**:
  - **Terminal Aesthetics**: A state-of-the-art glowing neon interface with animated glitches.
  - **Dynamic Dashboard**: A sleek sidebar layout separating Active Elections, the Admin Panel, and Configuration. 
  - **Smart Segregation**: The Admin Panel is strictly hidden from standard voters and dynamically unlocks only if your wallet matches the contract owner.

## 🤖 Deployment Automation & Bash Script Integration (The "Magic")

A standout feature of this project is the **fully automated contract address synchronization**. You never have to manually copy a deployed contract address from the terminal to your frontend code. 

Here is a bit-by-bit breakdown of exactly how this automated workflow functions:

1. **Log Generation (`Makefile`)**: 
   When you trigger a deployment command like `make deploy-anvil`, the `Makefile` intercepts the terminal output generated by Foundry using the `tee` command. These raw deployment logs are saved directly into the newly created `contract-deploy-logs/` directory.

2. **Script Trigger**: 
   The very next command in the `Makefile` triggers the custom bash script located at `bashScript/extract_address.sh`, passing the new log file as an argument.

3. **Pattern Matching & Extraction**: 
   The Bash script scans the log file using the `grep` command with a regular expression (`-Eo '0x[a-fA-F0-9]{40}'`). It specifically hunts for the exact 40-character hexadecimal string that represents your newly deployed smart contract address.

4. **Dynamic JavaScript Generation**: 
   Once the address is isolated, the script creates (or overwrites) a file in the project root named `contractaddress.js`. It physically writes the Javascript code inside this file: `const CONTRACT_ADDRESS = "0xYourNewAddress...";`.

5. **Frontend Injection**: 
   In `app.html`, there is a line `<script src="contractaddress.js"></script>`. This loads the globally defined address directly into the browser the moment you refresh the page.

6. **Instant Auto-Connection**: 
   Finally, `app.js` detects the presence of the `CONTRACT_ADDRESS` variable. It auto-fills the "Target Contract Address" input field and instantly establishes the connection between your Web3 interface and the blockchain!

## 🛠 Prerequisites

- [Foundry](https://getfoundry.sh/) (Forge, Anvil, Cast)
- [Node.js](https://nodejs.org/) or Python (for spinning up a local HTTP server)
- MetaMask Extension installed in your browser.

## ⚙️ Setup & Configuration

1. **Environment Variables:**
   Copy the example environment file and fill in your network and wallet keys.
   ```bash
   cp .env.example .env
   ```
   *Note: Update the `.env` file with your specific `SEPOLIA_RPC_URL`, `ETHERSCAN_API_KEY`, and `ANVIL_PRIVATE_KEY`.*

## 🚀 Commands (Makefile)

- **Clean the project:**
  Clears the forge cache, out directories, and previously stored deployment logs.
  ```bash
  make clean
  ```

- **Build the smart contracts:**
  ```bash
  make build
  ```

- **Run Tests:**
  Runs the Foundry test suites with verbose output `-vvv`.
  ```bash
  make test
  ```

- **Deploy to local Anvil node:**
  1. Start anvil in a separate terminal:
     ```bash
     anvil
     ```
  2. Run the deployment command:
     ```bash
     make deploy-anvil
     ```
  *(This triggers the automation script to write `contractaddress.js`)*

- **Deploy to Sepolia Testnet:**
  This command will interactively ask for your private key (`-i 1`), keeping it completely safe from your terminal history, before triggering the automation script.
  ```bash
  make deploy-sepolia
  ```

## 💻 Frontend Interaction

1. Deploy your contract using either Anvil or Sepolia as instructed above. (This automatically populates the `contractaddress.js` file with your deployment address).
2. Start a local static server in your project root to avoid CORS restrictions with MetaMask:
   ```bash
   # Using Python 3
   python3 -m http.server 8000
   ```
   Or using Node.js:
   ```bash
   npx serve .
   ```
3. Navigate to `http://localhost:8000/index.html` in your browser.
4. Click **INITIATE VOTE** to enter the main app console (`app.html`).
5. Click **CONNECT WALLET** in the sidebar. Once linked, the platform will verify your identity.
6. The contract address is already loaded! Click **CONNECT CONTRACT** if it hasn't auto-synced.
7. If your connected wallet is the deployer, the hidden **ADMIN PANEL** tab will reveal itself, allowing you to instantly initialize new elections.
8. Navigate to the **DASHBOARD** to view active ledgers and securely cast your vote.
